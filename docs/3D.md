# 3D / WebGL module

Optional **project-local** stack for React Three Fiber (R3F) and WebGL work. Separate from the 2D frontend module — Impeccable is for dashboards, not scene graphs.

## Diagnosis (why 3D attempts fail in Cursor)

Forensics on the machine that built v0.2.0:

| Check | Result |
|-------|--------|
| R3F / `three` deps in `Projects/` | **None found** |
| Session archives mentioning 3D work | **None** |
| v0.2 frontend stack coverage | 2D SaaS only (Impeccable + ui-ux-pro-max + dashboard design-refs) |
| ui-ux-pro-max implementation stacks (pre-v0.3) | `react-tailwind`, `react-native` only — no R3F CSV |

**Conclusion:** Failures are structural, not a single bad file. The agent had aesthetic hints ("use Three.js") but no implementation lane, proof gate, or stack-specific rules. Work was likely ephemeral (uncommitted chat) or applied the wrong module (2D Impeccable on WebGL).

### Typical failure modes (without this module)

| Symptom | Root cause | Fix |
|---------|------------|-----|
| Black canvas | No lights, camera inside mesh, wrong scale | Proof scene gate (box + lights + orbit first) |
| SSR crash / `window is not defined` | Canvas on server | `'use client'` + `dynamic(..., { ssr: false })` |
| Blank page, no errors | Canvas height 0 | Explicit `h-screen` / `width:100%; height:100%` on wrapper |
| Janky / mobile melt | Full DPR, huge textures | `dpr={[1, 2]}`, Draco, texture caps |
| "Environment" never ships | Scope too big day one | Milestones: proof -> props -> lighting -> assets |
| Impeccable fights the work | Purple-gradient / flat UI detectors | Use **3D lane**, not `/impeccable polish` on WebGL |

## Install (3D / R3F repo)

From your project root:

```powershell
# Windows
& "C:\path\to\cursor-agent-stack\project-template\install-3d.ps1"
```

```bash
# macOS / Linux
chmod +x /path/to/cursor-agent-stack/project-template/install-3d.sh
/path/to/cursor-agent-stack/project-template/install-3d.sh
```

Installs:

- `.cursor/skills/r3f-three/` + `ui-ux-pro-max/` (stack lookup)
- `.cursor/rules/3d-interactive-lane.mdc`
- `.cursor/design-refs/3d.md`
- `scenes/ProofScene.tsx` (if missing)

Does **not** install Impeccable. For hybrid apps (dashboard + 3D hero), run **both** installers.

## After install

1. **Reload Cursor** — enable third-party agent configs.
2. Add deps: `three`, `@react-three/fiber`, `@react-three/drei`.
3. Ask agent to **render ProofScene** before any environment work.
4. Lookup: `python .cursor/skills/ui-ux-pro-max/scripts/search.py "lights canvas" --stack react-three-fiber`

## Stack picker

| Goal | Stack | Notes |
|------|-------|-------|
| Dashboard, forms, tables | 2D frontend module | Impeccable |
| Product spin / GLB viewer | model-viewer or Spline embed | Not full R3F |
| Interactive scene | R3F + drei | This module |
| CSS parallax | Tailwind transforms | No WebGL |

## Milestones (mandatory order)

1. **Proof** — mesh + lights + OrbitControls
2. **Stage** — ground, grid, Environment/HDRI
3. **Content** — GLTF + Draco
4. **Polish** — postprocessing after 60fps proof
